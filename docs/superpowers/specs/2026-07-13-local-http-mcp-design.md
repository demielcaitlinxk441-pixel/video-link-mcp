# 本机 HTTP MCP 设计

## 目标

在不影响现有 stdio MCP 使用方式的前提下，为 Video Link Analyzer 增加本机 HTTP MCP 接入方式，让支持 Streamable HTTP MCP 的客户端能够连接。

## 范围

- 保留 `python server.py` 的既有 stdio MCP 行为。
- 新增命令行参数，显式启动 HTTP MCP。
- HTTP 服务仅监听 `127.0.0.1`，默认端口为 `8000`。
- 提供端口覆盖参数，供同一电脑上存在端口冲突时使用。
- 更新配置示例、安装提示和 README。
- 为启动参数和本机监听行为补充自动化测试。

## 不包含的内容

- 不开放 `0.0.0.0`、局域网或公网访问。
- 不新增登录、鉴权、反向代理或云端部署。
- 不改变五个 MCP 工具的名称、参数或返回格式。

## 架构

服务仍由 `server.py` 中的同一个 FastMCP 实例提供。入口根据命令行参数选择传输方式：未指定参数时使用 stdio；指定 HTTP 参数时使用 Streamable HTTP。HTTP 模式始终传入 `127.0.0.1`，从而只接受当前电脑的连接。

HTTP URL、路径与传输参数以已安装的 MCP Python SDK 实际行为为准，并在实现时通过自动化启动测试验证。文档将只展示经验证可用的 URL。

## 使用方式

```powershell
# 保持现有行为：由 MCP 客户端通过标准输入输出启动
python server.py

# 启动仅供本机访问的 HTTP MCP
python server.py --transport http

# 本机端口冲突时，改用 8765 端口
python server.py --transport http --port 8765
```

每位使用者在自己的电脑上克隆、安装并启动服务后，均可访问自己的 `127.0.0.1` 地址；不同电脑之间不会互相访问。

## 错误处理

- 对不支持的传输参数给出清晰的命令行错误。
- 端口被占用时由 HTTP 服务器报告启动失败；README 说明使用 `--port` 更换端口。
- stdio 模式不输出额外的标准输出日志，避免干扰 MCP 通信。

## 验证

- 新测试验证默认选择 stdio、HTTP 模式固定主机为 `127.0.0.1`、端口默认值和覆盖值。
- 现有完整单元测试继续通过。
- 使用本机 HTTP MCP 客户端请求初始化与工具列表，确认真实连通性。
- 运行项目离线自检脚本。
